服务器部署hexo

由于国内访问 hc1998.github.io 的速度太慢，于是决定将博客部署至服务器

hexo在服务器端部署需要：

  1. 安装环境：包括node.js、git库等环境
  2. 本地安装hexo
  3. ssh连接（本地创建ssh公钥，添加至服务器端）
  4. 博客自动更新脚本

本地环境部署：

  1. 安装hexo环境
  2. 安装git bash工具
  3. 修改_config.yml文件中的repository改至自己的博客地址
deploy:
  type: git
  repository: git@github.com:hc1998/hc1998.github.io.git
  branch: master
  1. hexo clean 清除缓存

    hexo g 生成静态网页

    hexo d 进行部署

总结部署至服务器失败原因：

  • 由于购买的vps是 CentOS 6.x 的系统，自带的 gcc 版本为v4.4.7，不足以支撑 node.js 的安装。所以先进行了一次升级，升级至v4.9.0版本，升级时间大概为1小时，之后安装 node.js ，出现未知错误，导致编译失败，查了很久也没查出原因，故先告一段落。

  • 在更换服务器系统之前，安装好所有环境进行调试的过程中，ssh请求失败。无法将数据提交给服务器建好的git库，导致数据无法上传。

  • 检测过后发现服务器端ssh 22端口未打开，打开后问题依然存在。上网搜索后将空密码ssh公钥改成有密码公钥，依然未生效。
